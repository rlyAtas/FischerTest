# Пользователи и авторизация

## Регистрация и вход

Используется единый способ авторизации: вход по `email` с одноразовым кодом подтверждения.

### Общий принцип

Пользователь вводит свой email, получает одноразовый код на почту, вводит его в приложении и после успешной проверки получает доступ к аккаунту.

Если пользователь входит с новым email впервые:
- создается новый аккаунт;
- пользователь автоматически авторизуется.

Если пользователь вводит email, для которого аккаунт уже существует:
- система не создает новый аккаунт;
- пользователь входит в существующий аккаунт после подтверждения кода.

Таким образом, регистрация и вход объединены в один сценарий.

### Цель

Такой способ входа нужен для того, чтобы:
- упростить старт работы с сервисом;
- убрать необходимость запоминать пароль;
- сократить количество барьеров при первом использовании.

### Сценарий входа

1. Пользователь открывает экран входа.
2. Вводит email.
3. Система отправляет одноразовый код на указанный адрес.
4. Пользователь вводит код в приложении.
5. Система проверяет код.
6. Если код верный и не истек, пользователь авторизуется.
7. Если email новый, создается новый аккаунт.
8. Если email уже существует, пользователь входит в существующий аккаунт.

### Правила одноразового кода

- код состоит из 6 цифр;
- код действует ограниченное время, 10 минут;
- код можно использовать только один раз;
- количество попыток ввода кода ограничено тремя;
- повторная отправка кода доступна с небольшой задержкой, через 60 секунд.

### Сессия пользователя

После успешного входа пользователь получает авторизованную сессию. Пока сессия активна, пользователь может работать с приложением без повторного ввода кода. После завершения сессии пользователь должен заново подтвердить вход через email-код.

Сессия должна сохраняться в течение ограниченного времени:
- до ручного выхода из аккаунта;
- или до истечения срока действия сессии.

### Ошибки и сообщения

Система должна корректно обрабатывать следующие ситуации:
- email введен в неверном формате;
- код неверный;
- код истек;
- превышено число попыток ввода;
- повторная отправка пока недоступна;
- письмо не удалось отправить;
- произошла временная ошибка сервиса.

### Выход из аккаунта

При выходе:
- авторизованная сессия завершается;
- пользователь теряет доступ к серверным данным аккаунта и работе с приложением.

### Удаление аккаунта

Удаление аккаунта в MVP не выполняется пользователем самостоятельно через интерфейс.

Пользователь может запросить удаление аккаунта и связанных персональных данных через контактный email, указанный в Privacy Policy.

После подтверждения запроса администратор удаляет персональные данные пользователя и отзывает активные сессии.

## Жизненный цикл зарегистрированного пользователя

### Кто такой зарегистрированный пользователь

Зарегистрированный пользователь — это пользователь, у которого есть аккаунт в системе и активная или ранее созданная авторизованная учетная запись. Аккаунт создается автоматически после успешного подтверждения `email` одноразовым кодом.

### Цель зарегистрированного режима

Режим зарегистрированного пользователя нужен для того, чтобы:
- сохранять прогресс на сервере;
- работать с приложением на разных устройствах;
- не терять данные при очистке браузера;
- хранить историю тренировок, экзаменов и избранного;
- продолжать обучение с того места, где пользователь остановился.

### Начало жизненного цикла

Жизненный цикл зарегистрированного пользователя начинается, когда пользователь:
- вводит свой `email`;
- получает одноразовый код;
- подтверждает код;
- успешно проходит авторизацию.

Если пользователь входит с новым `email` впервые:
- система создает новый аккаунт;
- пользователь становится зарегистрированным пользователем.

Если пользователь входит с уже существующим `email`:
- система не создает новый аккаунт;
- пользователь входит в ранее созданный аккаунт;
- загружаются серверные данные пользователя.

### Что хранится у зарегистрированного пользователя

Для зарегистрированного пользователя на сервере хранятся:
- `email`;
- язык интерфейса;
- выбранная федеральная земля;
- история тренировок;
- результаты ответов по вопросам;
- статусы вопросов:
  - `problematic`
  - `learning`
  - `mastered`
- избранные вопросы;
- вероятность подготовки к экзамену;
- технические данные авторизации и сессий.

### Возможности зарегистрированного пользователя

Зарегистрированный пользователь может:
- проходить тренировки по темам;
- проходить тренировки по избранным вопросам;
- проходить эмуляцию экзамена;
- сохранять и восстанавливать прогресс;
- использовать приложение на нескольких устройствах;
- видеть историю тренировок и экзаменов;
- добавлять/удалять избранные вопросы;
- изменять язык интерфейса;
- изменять федеральную землю;
- возвращаться к обучению после перерыва без потери данных.

### Поведение внутри продукта

#### 1. Первый вход
Пользователь подтверждает `email` одноразовым кодом и получает авторизованную сессию.

#### 2. Создание аккаунта
Если `email` новый, система создает аккаунт и связывает его с этим адресом.

#### 3. Основное использование
Пользователь проходит тренировки и экзамены, а система после каждого значимого действия обновляет серверные данные:
- ответы на вопросы;
- статусы вопросов;
- избранное;
- результаты экзаменов;
- прогресс.

#### 4. Возврат в приложение
При повторном входе с того же или другого устройства пользователь получает доступ к своим сохраненным данным аккаунта.

### Сессия пользователя

После успешного входа пользователь получает авторизованную сессию.

Пока сессия активна:
- повторный ввод кода не требуется;
- пользователь работает с приложением как авторизованный пользователь.

После завершения сессии:
- пользователь теряет доступ к серверным данным до повторной авторизации;
- для нового входа снова используется подтверждение через `email` и одноразовый код.

### Изменение настроек

Зарегистрированный пользователь может изменять:
- язык интерфейса;
- федеральную землю;
- иные настройки профиля, если они будут добавлены позже.

Изменения сохраняются в аккаунте и применяются при следующем входе.

### Работа на нескольких устройствах

Так как данные зарегистрированного пользователя хранятся на сервере:
- пользователь может войти с другого устройства;
- после входа он получает доступ к своему актуальному прогрессу;
- серверные данные являются основным источником истины.

### Выход из аккаунта

При выходе:
- текущая авторизованная сессия завершается;
- данные аккаунта не удаляются;
- при повторном входе данные аккаунта снова становятся доступны.

### Завершение жизненного цикла

Жизненный цикл зарегистрированного пользователя может быть завершен, если:
- аккаунт удаляется администратором по запросу пользователя или в иных предусмотренных системой случаях;
- пользователь перестает пользоваться приложением, но аккаунт при этом остается существовать.

Если аккаунт удален:
- пользователь теряет доступ к сохраненному серверному прогрессу.

### Важные правила

- Один `email` может быть связан только с одним аккаунтом.
- Авторизация и вход выполняются без пароля, только через одноразовый код.
- Все учебные данные зарегистрированного пользователя должны сохраняться на сервере автоматически.
