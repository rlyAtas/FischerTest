# Бизнес-логика

## Общая информация

Документ описывает бизнес-логику приложения, связанную с:
- организацией вопросов
- статусами вопросов
- режимами тренировки
- алгоритмом подбора вопросов
- работой пользователя

## Вопросы

Каждый вопрос:
- принадлежит только одной **федеральной земле**;
- принадлежит только одной **теме**;
- может быть добавлен пользователем в **избранное**.

### Темы вопросов

Тема вопроса принадлежит конкретной федеральной земле.

Название темы хранится в оригинальном немецком виде. Для отображения темы на языке пользователя могут использоваться переводы темы.

### Статусы вопросов

Статус вопроса отражает текущий прогресс пользователя по конкретному вопросу.

Статус **new** означает, что пользователь ещё не отвечал на вопрос. Он не хранится в базе данных и определяется отсутствием записи о прогрессе пользователя по данному вопросу.

Остальные статусы хранятся в прогрессе пользователя:

- **problematic** — последний ответ пользователя на вопрос был неправильным;
- **learning** — пользователь правильно ответил на вопрос после статуса **new** или **problematic**;
- **mastered** — пользователь правильно ответил на вопрос после статуса **learning** или повторно правильно ответил на вопрос в статусе **mastered**.

#### Правило перехода статусов после ответа:

| Текущий статус | Ответ | Новый статус |
|---|---|---|
| **new** | правильный | **learning** |
| **new** | неправильный | **problematic** |
| **problematic** | правильный | **learning** |
| **problematic** | неправильный | **problematic** |
| **learning** | правильный | **mastered** |
| **learning** | неправильный | **problematic** |
| **mastered** | правильный | **mastered** |
| **mastered** | неправильный | **problematic** |

## Тренировки

Существует два режима тренировки:
1. Показ вопросов подряд. Используется для ознакомления с вопросами.
2. Эмуляция экзамена.

#### Показ всех вопросов подряд 

Варианты формирования набора вопросов:
- **all** — все вопросы выбранной федеральной земли;
- **topic** — вопросы выбранной темы;
- **favorites** — избранные вопросы.

Поведение:
- все вопросы показываются последовательно один за одним, порядок выдачи вопросов: **new**, **problematic**, **learning**, **mastered**;
- пользователь видит статус текущей тренировки: общее количество вопросов и количество отвеченных вопросов;
- пользователь может получить перевод вопросов и ответов на язык интерфейса приложения;
- после ответа пользователя на вопрос пользователь видит сразу правильный ответ;
- после ответа пользователя на вопрос обновляется прогресс пользователя по вопросу (item_progress);
- пользователь не может изменить ответ;
- пользователь может переходить к следующему/предыдущему вопросу;
- вопрос можно добавить в избранное или удалить из избранного;
- пользователь может закончить тренировку в любой момент;
- неотвеченные вопросы после окончания тренировки игнорируются для оценки прогресса пользователя по вопросам (item_progress);
- общий прогресс пользователя (user_progress) пересчитывается после завершения тренировки по полученным ответам.

#### Тренировка по избранным вопросам

Тренировка по избранным вопросам может быть создана только если у пользователя есть хотя бы один избранный вопрос для выбранной федеральной земли.

Если избранных вопросов нет, система не создаёт тренировку и возвращает доменную ошибку `empty_favorites_training`.

#### Эмуляция экзамена

Формирование набора вопросов по правилам реального экзамена.
Подробности в [правилах экзамена](./exam-rules.md).

Поведение:
- пользователь видит статус текущей эмуляции: общее количество вопросов и количество отвеченных вопросов;
- пользователь может переходить к следующему/предыдущему вопросу;
- пользователь может изменить ответ;
- вопрос можно добавить в избранное или удалить из избранного;
- пользователь может закончить эмуляцию в любой момент;
- после завершения эмуляции пользователь видит результат эмуляции и пояснение причины провала;
- неотвеченные вопросы влияют только на результат текущей эмуляции, они засчитываются как неправильные;
- неотвеченные вопросы не влияют на прогресс пользователя по вопросам;
- прогресс пользователя по вопросам (item_progress) обновляется после завершения эмуляции по полученным ответам;
- общий прогресс пользователя (user_progress) пересчитывается после завершения эмуляции по полученным ответам.

## Пользователь

Пользователь в системе идентифицируется с помощью email.

Прогресс пользователя рассчитывается отдельно для выбранной федеральной земли.

### Прогресс пользователя

Этот показатель является по сути целью приложения.

Прогресс представлен двумя параметрами:
1. четырехцветная диаграмма, показывающая количество вопросов в каждом статусе;
2. вероятность успешной сдачи экзамена.

### Оценка прогресса пользователя в виде диаграммы

Для расчета диаграммы система получает актуальное количество вопросов для каждого статуса. На основе этого строится четырехцветная диаграмма.

Диаграмма отображается в следующих цветах:
- зеленый для **mastered**;
- желтый для **learning**;
- красный для **problematic**;
- серый для **new**.

Диаграмма не хранится как отдельная сущность или поле в `user_progress`.

Для отображения диаграммы система каждый раз рассчитывает количество вопросов в статусах **new**, **problematic**, **learning** и **mastered** на основе:
- полного списка вопросов выбранной федеральной земли;
- записей `item_progress` пользователя по этим вопросам.

Статус **new** определяется отсутствием записи в `item_progress`.


### Оценка вероятности успешной сдачи экзамена

Вероятность сдачи экзамена является оценочным показателем. Она не гарантирует результат реального экзамена, а показывает текущий уровень готовности пользователя на основе его истории ответов.

Для расчета вероятности каждому статусу вопроса сопоставляется вероятность правильного ответа:
- **mastered** — **100%**;
- **learning** — **50%**;
- **problematic** — **0%**;
- **new** — **0%**.

Расчет выполняется методом Монте-Карло. Для этого система:
1. получает массив всех вопросов выбранной федеральной земли вместе с их текущими статусами;
2. **X** раз моделирует экзамен по правилам выбранной федеральной земли;
3. в каждой симуляции формирует экзаменационный пул по действующим правилам отбора вопросов;
4. для каждого вопроса в пуле случайным образом определяет, ответит ли пользователь правильно, на основе вероятности, соответствующей статусу вопроса;
5. подсчитывает результат симуляции и определяет, считается ли экзамен сданным согласно [правилам экзамена](./exam-rules.md);
6. после завершения всех симуляций рассчитывает долю успешных экзаменов.

Средняя вероятность сдачи экзамена рассчитывается по формуле:
**probability = successful_simulations / X * 100**

Итоговая вероятность показывается пользователю в процентах, с округлением до целого значения.

## Переводы

Оригинальный текст тем, вопросов и вариантов ответов хранится на немецком языке и всегда показывается пользователю.

Немецкий текст является основным учебным и экзаменационным текстом, потому что реальный экзамен проходит на немецком языке.

Переводы тем, вопросов и вариантов ответов используются только как вспомогательный учебный инструмент.

В обычных тренировках (all, topic, favorites) пользователь может запросить перевод вопроса и вариантов ответов на выбранный язык пользователя.

Перевод не заменяет немецкий текст, не влияет на проверку ответа и не используется как основной текст вопроса.

В режиме эмуляции экзамена вопрос и варианты ответов должны оставаться на немецком языке.
